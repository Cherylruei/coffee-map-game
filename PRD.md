## 1. 專案概述

### 1.1 專案背景

**咖啡地圖收集**是一個結合訂單管理與遊戲化行銷的咖啡社團營運系統。透過收集咖啡產地卡片的方式，促進咖啡銷量並帶動顧客認識不同咖啡豆品種，同時藉由 LINE 分享與交換卡片功能，創造話題並拓展新客群。

### 1.2 問題定義

#### 現況痛點

| 問題             | 影響                             | 嚴重度  |
| ---------------- | -------------------------------- | ------- |
| 紙本點單易出錯   | 店員需要重新確認、浪費時間       | 🔴 高   |
| 無法追蹤銷售數據 | 不知道哪款咖啡熱銷、誰收了多少款 | 🟡 中   |
| 手動計算金額     | 容易算錯、找錯錢                 | 🔴 高   |
| 缺乏顧客互動機制 | 顧客忠誠度低、回購率難提升       | 🟡 中   |
| 原料管理混亂     | 不知道用量、無法預測補貨         | 🟠 中高 |

### 1.3 解決方案

#### 系統核心價值

```
商家後台（店員使用）
├─ 數位化點單：取代紙本，減少錯誤
├─ 智慧計算：自動算總額、折扣、客製化加價
├─ 收款記錄：追蹤付款方式與收款人
├─ 訂單管理：可修改、可查詢、可統計
├─ 修改菜單: 可同步修改商家後台與前台菜單
├─ 原料盤點：每日記錄用量、預測補貨
└─ 數據分析：營收、熱銷品項、店員績效

消費者前台（顧客使用）
├─ LINE Login：快速登入，無需註冊
├─ 菜單瀏覽：查看品項、價格、售完狀態（唯讀）
├─ QR Code 抽卡：購買咖啡即可收集卡片
├─ 咖啡地圖：12 張產地卡片，稀有度分級
├─ 分享功能：與好友交換卡片
└─ 獎勵機制：集滿送咖啡券、累積送折扣
```

### 1.4 成功指標 (KPIs)

#### 短期目標（3個月內）

- ✅ 系統上線並穩定運作
- ✅ 店員熟悉操作，點單時間減少 50%
- ✅ 至少 60% 顧客註冊 LINE 參與遊戲
- ✅ 每日營業數據自動統計

#### 中長期目標（6個月-1年）

- 📈 顧客回購率提升 30%
- 📈 平均客單價提升 15%（透過客製化、折扣券促銷）
- 📈 至少 20% 顧客集滿 12 張卡片
- 📈 分享卡片功能使用率達 40%
- 📈 維持穩定的社團氛圍，保持休閒與社交屬性

## 2. 用戶分析

### 2.1 目標用戶

#### 主要用戶角色

| 角色              | 人數       | 使用場景         | 核心需求                     |
| ----------------- | ---------- | ---------------- | ---------------------------- |
| **店員**          | 3-5人      | 營業時間操作後台 | 快速點單、收款記錄、訂單管理 |
| **咖啡社員/常客** | 15-20人/日 | 每週購買咖啡     | 收集卡片、獲得獎勵           |
| **一般顧客**      | 5-10人/日  | 偶爾購買         | 簡單購買流程                 |

**營運特性補充：**

- 營業時間：每週一、三中午 12:00-13:00（各 1 小時）
- 日均客量：15-20 人，旺季可達 20-30 杯
- 社團性質：以休閒、社交為主，營收為輔
- 環保政策：**顧客統一使用自帶杯**（無提供紙杯）

#### 用戶畫像

**店員 - Alice**

- **基本資訊**：25歲，咖啡社團幹部，熟悉咖啡知識
- **使用情境**：每週一、三中午負責點單、收款、製作咖啡
- **痛點**：紙本記錄容易出錯、手算金額慢、不知道庫存
- **期望**：快速點單、自動計算、方便修改、清楚統計

**顧客 - Bob**

- **基本資訊**：28歲，公司員工，每週買 2-3 杯咖啡
- **使用情境**：中午到櫃台買咖啡，順便聊天
- **痛點**：有時想點的咖啡賣完了、不知道有什麼優惠
- **期望**：能看到菜單、知道有什麼活動、收集卡片有趣

### 2.2 使用場景

#### 場景 1：尖峰時段快速點單

```
時間：週一中午 12:10
地點：公司咖啡櫃台
人物：店員 Alice、顧客 Bob、Carol、David（排隊中）

Bob：「我要一杯冰拿鐵，加 Double Shot」
Alice：
  1. 開啟後台點單
  2. 選擇「冰拿鐵」
  3. 加入「Double Shot +$15」
  4. 顯示總額「$60」
  5. Bob 掃 LINE Pay 付款
  6. Alice 點選「LINE Pay」並確認
  7. 系統生成 QR Code
  8. Bob 掃碼抽卡，獲得「肯亞AA」

時間：約 30 秒完成
```

#### 場景 2：使用折扣券

```
時間：週三中午 12:20
人物：店員 Alice、顧客 Carol（集滿 12 張卡）

Carol：「我集滿了！要用咖啡券換一杯拿鐵」
Alice：
  1. 點單：熱拿鐵 $45
  2. 輸入折扣券：45（減 $45）
  3. 總額：$0
  4. 確認訂單（系統記錄券已使用）
  5. 生成 QR Code
  6. Carol 掃碼（記錄本次消費）
```

#### 場景 3：訂單修改

```
時間：週一中午 12:15
人物：店員 Alice、顧客 David

David：「我剛才點的冰美式可以改成熱的嗎？」
Alice：
  1. 進入「訂單管理」
  2. 找到 David 的訂單（20260317-001）
  3. 點擊「修改」
  4. 將「冰美式」改成「熱美式」
  5. 確認修改
  6. 重新製作咖啡
```

#### 場景 4：營業結束盤點

```
時間：週一 13:10（營業結束後）
人物：店員 Alice

Alice：
  1. 點擊「結束營業」
  2. 系統顯示當日訂單統計
  3. 輸入盤點資料：
     - 咖啡豆剩餘：1 包 500 克
     - 牛奶剩餘：2 瓶 1800 ml
  4. 確認送出
  5. 系統保存並計算用量
  6. 提示下次建議補貨量
```

## 9. 風險與限制

### 9.1 技術風險

| 風險                 | 影響  | 機率  | 對策                           |
| -------------------- | ----- | ----- | ------------------------------ |

| 資料庫效能問題       | 🟡 低 | 🟢 低 | 索引優化，定期監控             |
| 並發訂單處理         | 🟡 低 | 🟢 低 | 使用事務，樂觀鎖               |

### 9.2 業務風險

| 風險             | 影響  | 機率  | 對策                   |
| ---------------- | ----- | ----- | ---------------------- |
| 店員不適應新系統 | 🟠 中 | 🟡 中 | 充分培訓，保留紙本備案 |
| 顧客不參與遊戲   | 🟡 低 | 🟡 中 | 優化遊戲體驗，加強宣傳 |

- 系統設計簡單直觀
- 避免過度複雜功能
- 保持小而美的定位

- 顧客統一自帶杯
- 無法提供外送服務
- 不適合線上預訂

**因應策略：**

- 取消線上點單功能
- 提供唯讀菜單瀏覽
- 強化現場服務體驗

| `customization_added` | 加入客製化   | customization_type: 'double_shot'        | 客製化使用率 |
| `discount_applied`    | 使用折扣券   | discount_amount                          | 折扣券效益   |
| `daily_close`         | 完成每日盤點 | total_revenue, total_orders              | 營運統計     |
| `menu_soldout`        | 標記售完     | item_id                                  | 缺貨頻率     |

#### 10.1.3 關鍵指標 (KPIs)

**用戶指標**

```
DAU (Daily Active Users)
- 定義：每日活躍用戶數
- 計算：當天有執行任何事件的用戶數
- 目標：穩定成長

MAU (Monthly Active Users)
- 定義：每月活躍用戶數
- 目標：20-30 人穩定參與

登入率
- 計算：登入用戶數 / 總訪客數
- 目標：> 60%

回訪率
- 計算：回訪用戶數 / 總用戶數
- 目標：> 40% (每週)
```

**遊戲指標**

```
抽卡轉換率
- 計算：掃描QR用戶數 / 購買咖啡用戶數
- 目標：> 80%

平均收集進度
- 計算：平均每人收集卡片數
- 目標：5 張/人（3個月內）

集滿率
- 計算：集滿12張用戶數 / 總用戶數
- 目標：> 20%（6個月內）

分享使用率
- 計算：使用分享功能用戶數 / 總用戶數
- 目標：> 40%
```

**銷售指標**

```
客單價
- 計算：總營收 / 訂單數
- 目標：> $50

客製化率
- 計算：含客製化訂單數 / 總訂單數
- 目標：> 30%

折扣券使用率
- 計算：使用折扣券訂單數 / 總訂單數
- 監控：避免濫用

複購率
- 計算：購買2次以上用戶數 / 總用戶數
- 目標：> 60%
```

**營運指標**

```
點單效率
- 計算：平均點單完成時間
- 目標：< 1分鐘/單

訂單修改率
- 計算：修改訂單數 / 總訂單數
- 目標：< 10%

QR 掃描成功率
- 計算：成功掃描數 / 總掃描嘗試數
- 目標：> 95%

售完頻率
- 追蹤：哪些品項經常售完
- 用途：優化備貨
```

| **員編**     | 公司員工編號，用於未登入顧客的身份記錄                 |
