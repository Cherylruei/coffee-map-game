## 🎯 開發原則

### 介面設計規則

1. **已開發頁面**：保持現有設計，僅優化（需確認）
2. **未開發頁面**：按照 PRD 第 5 章介面設計開發
3. **色系**：深色咖啡色系（#2c1810, #6f4e37, #d4af37, #f5ebe0）

### 程式碼規範

- React Hooks（函式組件）
- async/await（非 Promise.then）
- 錯誤處理要完整
- 註解要清楚

### 測試要求

- 每個新功能都要手動測試
- 邊界情況要考慮（例如：折扣券大於訂單金額）

## 📌 重要注意事項

### 業務邏輯

1. **自帶杯政策**：不開放線上自助點單
2. **LINE Pay**：店員個人收款，系統僅記錄
3. **折扣券**：先做直接輸入金額，券碼系統未來再做
4. **客製化**：目前只有 Double Shot，一杯限一個

### 資料記錄

- 每筆訂單必須記錄收款店員
- QR Code 必須連結訂單資訊
- 未登入顧客用員編記錄
